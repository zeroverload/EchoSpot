🌟 项目简介
EchoSpot（回响坐标） 是一个对标大众点评的高并发本地生活服务平台。项目采用前后端分离架构，基于 Redis 集群 与 Spring Boot 构建，旨在解决高并发场景下的热点数据查询、秒杀抢购及社交互动等核心痛点。

本项目不仅仅是一个功能实现的集合，更是一次对高性能、高可用架构设计的深度探索。通过对 Redis 的深度定制化使用，成功构建了包含用户认证、商户检索、秒杀营销、社交图谱、LBS 位置服务在内的完整业务闭环。

🚀 核心亮点
高性能架构：通过 Redis 多级缓存策略，大幅降低数据库压力，支撑高并发流量冲击。
分布式一致性：巧妙运用分布式锁与乐观锁机制，严格保障“一人一单”与“库存扣减”的数据一致性。
异步解耦：引入 Redis Stream 消息队列，实现秒杀请求的异步削峰填谷，显著提升系统吞吐量。
LBS 地理服务：基于 Redis GEO 哈希算法，实现高效的“附近商户”实时检索功能。

🏗️ 技术架构
项目采用主流的微服务架构思想设计，关键技术栈如下：

核心框架：Spring Boot + MyBatis-Plus
中间件：Redis (Jedis/Lettuce) + Nginx + RabbitMQ (可选)
数据存储：MySQL (主从集群) + Redis (Cluster)
工具库：Hutool + Lombok + Redisson

🔧 技术攻坚与实现细节
1. 分布式会话与鉴权体系
   痛点解决：摒弃传统 Session 模式，基于 Redis 实现分布式 Token 存储，完美解决了多 Tomcat 节点下的 Session 共享与一致性问题。
   拦截器设计：自定义 HandlerInterceptor，实现对用户登录状态的自动化校验与 Token 有效期的无感刷新，提升用户体验与系统安全性。
2. 多级缓存与一致性方案
   缓存策略：采用 Cache Aside Pattern，配合缓存空对象策略，有效解决缓存穿透问题；通过逻辑过期机制解决缓存击穿问题，保障服务高可用。
   一致性保障：通过 Redis Lua 脚本封装复杂逻辑，确保缓存更新与数据库操作的原子性。
3. 秒杀系统设计
   超卖问题：利用数据库乐观锁（CAS 思想）防止库存超卖，保障数据准确性。
   一人一单：基于 Redis Set 数据结构预判 + Redisson 分布式锁，确保在集群模式下的“一人一单”业务逻辑线程安全。
   异步优化：重构秒杀流程，引入 Redis Stream 作为消息队列，实现请求的异步处理，将下单响应时间降低至毫秒级。
4. 社交与 LBS 功能
   点赞排行：利用 Redis ZSet 实现按时间/热度排序的点赞排行榜，支持快速 TopN 查询。
   社交图谱：基于 Set 集合实现用户关注、共同关注（交集运算）、好友推荐等社交功能。
   附近的人/店：应用 Redis GEO 数据结构存储商户经纬度，实现高效的 LBS（基于位置的服务）检索功能。
5. 用户行为分析
   BitMap 签到：利用 Redis BitMap 实现用户签到功能，以极低的内存占用（一个用户一年仅需约 46 字节）实现了高效的签到统计与连续签到判断。
   UV 统计：使用 HyperLogLog 进行页面独立访客统计，以极小的空间成本实现海量数据的去重计数。
   
📦 功能模块
   模块名称	核心功能	技术难点
   用户模块	短信登录、Session 共享、Token 刷新	Redis 分布式会话、拦截器
   商户模块	商户缓存查询、商户类型缓存	缓存穿透/雪崩/击穿解决方案
   优惠模块	秒杀下单、优惠券抢购	乐观锁、分布式锁、一人一单
   位置模块	附近商户搜索	Redis GEO、GeoHash
   社交模块	点赞排行榜、关注/共同关注、签到	ZSet、Set、BitMap
   消息模块	异步秒杀下单	Redis Stream 消息队列
   
📝 总结
   EchoSpot 项目不仅实现了业务逻辑的闭环，更重点解决了互联网场景下的高并发、数据一致性与系统可用性三大难题。通过该项目，深入掌握了 Redis 在缓存、分布式锁、消息队列、地理位置计算及大数据统计等多个维度的实战应用，具备极高的学习与参考价值。
